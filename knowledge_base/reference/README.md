# reference/

External resource pointers — links to outside projects, tool directories, and curated lists that are useful to consult but don't belong in this knowledge base as deep technique documentation. Unlike every other `knowledge_base/` category, entries here are not required to ground claims in this project's own genre/technique files, since they're explicitly attributed to a named external source rather than presented as this project's own researched fact.

No dedicated schema. Use plain YAML frontmatter (`title`, `tags`, `source_url` at minimum).

Not consumed by a specific subagent — check here ad hoc when a producer or agent needs a tool this knowledge base doesn't document in depth yet, before falling back to unsourced general knowledge.
